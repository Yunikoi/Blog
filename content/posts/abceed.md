### 新完全语法第一课

#### 〜際（に）

**要点：** 相当于“〜の時”，但语气更正式，常用于公共场所的告示、说明书或商务场合。 

**接续：** 名词+の / 动词辞书形、た形

#### 〜に際して／〜にあたって

**要点：** 用于表示某个重大的、正式的活动或行为开始前。

#### 〜たとたん

**要点：** 表示 A 发生后，几乎同时发生了出乎意料的 B。 

**接续：** 必须接动词的 **「た形」**。后面不能接自己的意志、请求或命令。

#### 〜かと思うと

**要点：** 表示前一个动作刚结束，紧接着发生了出乎意料的变化。

**特点：** 强调**意外感**或**前后的剧烈对比**。

**限制：** **不能用于描述说话人自己的动作**。

#### 〜か〜ないかのうちに

**要点：** 表示 A 刚刚完成或几乎还没完成的瞬间，B 就紧接着发生了。

**特点：** 强调**时间的紧迫感**，仿佛 A 和 B 是同时发生的。

**接续：** 动词辞书形 + か + 动词ない形 + かのうちに。

### 新完全语法第二课

---

#### 1.～最中だ / ～最中に
- **中文意思**：正在……当中；在……过程中
- **接续**：名词+の / 动词ている形 + 最中だ
- **核心特点/使用限制**：
  1.  表示动作正在进行的状态；
  2.  多用于短时间动作，后句常接「意外发生的事」打断当前动作；
  3.  例句：田中さんは今考えごとをしている最中だから、じゃましないほうがいい。（田中现在正在思考，最好不要打扰他）

---

#### 2. ～うちに（A用法：趁着……）
- **中文意思**：趁着……（还没……的时候）
- **接续**：名词+の / 动词辞书形/ている/ない形 / イ形容词い形 / ナ形容词な形 + うちに
- **核心特点/使用限制**：
  1.  强调「趁时间/状态还没变化，赶紧做某事」；
  2.  后句必须接**意志性动作**（自己主动做的事）；
  3.  例句：忘れないうちに、カレンダーにメモしておこう。（趁还没忘，先记在日历上吧）

---

#### 3. ～うちに（B用法：在……期间/不知不觉……）
- **中文意思**：在……期间；不知不觉……
- **接续**：动词辞书形/ている/ない形 + うちに
- **核心特点/使用限制**：
  1.  表示在某段时间内，发生了自然的变化；
  2.  后句接**非意志性变化**（不受人控制的事）；
  3.  例句：知らないうちに、雨が降り始めていた。（不知不觉间，下起雨来了）

---

#### 4. ～ばかりだ / ～一方だ
- **中文意思**：越来越……；一味地……
- **接续**：动词辞书形 + ばかりだ / 一方だ
- **核心特点/使用限制**：
  1.  表示某一方向的变化持续发展；
  2.  「ばかりだ」多用于**负面趋势**（越来越糟），「一方だ」为中性；
  3.  例句：このごろは仕事が多くて残業が増えるばかりだ。（最近工作多，加班越来越多）

---

#### 5. ～（よ）うとしている
- **中文意思**：即将……；正要……
- **接续**：动词う・よう形 + としている
- **核心特点/使用限制**：
  1.  表示「某事马上就要发生」的瞬间状态，语气偏硬；
  2.  只能接**瞬间动词**（开始、开幕、完成等）；
  3.  例句：さあ、決勝戦が今、始まろうとしています。（好了，决赛现在即将开始）

---

#### 6. ～つつある
- **中文意思**：正在……；逐步……
- **接续**：动词ます形 + つつある
- **核心特点/使用限制**：
  1.  表示变化正在持续进行中，书面、正式语气；
  2.  多用于**渐进式变化**（变暖、发展、近代化等）；
  3.  例句：次第に暖かくなりつつあります。春はもうすぐです。（天气正逐渐变暖，春天马上就到了）

---

#### 7. ～つつ
- **中文意思**：一边……一边……；同时……
- **接续**：动词ます形 + つつ
- **核心特点/使用限制**：
  1.  表示两个动作同时进行，书面、正式语气；
  2.  前后句必须是**同一主语**；
  3.  例句：いろいろな体験を楽しみつつ、日本の生活に慣れていった。（一边享受各种体验，一边习惯了日本的生活）

##### 一、 最强后缀篇（接在词尾）

#### 1. 费用/金钱类（最易混淆）

## 4.6

### 新完全语法第三课

---
#### 1. ～てはじめて

**要点：** 经历前项事情/处于某种状态后，才发生了此前从未有过的新情况、产生了新认知，强调「前项是后项的触发条件」。
**接续：** 动词て形 + はじめて
**补充：** 后半句必须是「新事物发生、新想法产生、新状态实现」相关内容。

---
#### 2. ～上(で)
**要点：** 完成前项动作/准备的基础上，再进行后续的意志性行为，强调动作的先后顺序与前提性，语气正式。
**接续：** 动词た形 + 上で；名词 + の + 上(で)（名词接续可省略「で」）
**补充：** 前后主语必须一致，不可用于动作顺序为自然必然的场景。

---
#### 3. ～次第
**要点：** 前项事情一完成，就立刻执行后项动作，是偏生硬、正式的表达，多用于商务/公务场景。
**接续：** 动词ます形（去掉ます） + 次第
**补充：** 后半句多为说话人的希望、意向、委托，强调「瞬间触发、无延迟」。

---
#### 4. ～て以来・～てこのかた
**要点：** 从过去某一时点做了某事之后到现在，一直持续同一种状态，无中断；「～てこのかた」比「～て以来」更书面、强调持续感。
**接续：** 动词て形 + 以来 / このかた
**补充：** 只能搭配过去的时间点，后半句必须是「持续到现在的状态」，不能接未来内容。

---
#### 5. ～てからでないと・～てからでなければ
**要点：** 不先完成前项动作，就无法实现后项，强调「前项是后项的必要前提」。
**接续：** 动词て形 + からでないと / からでなければ
**补充：** 后半句必须是否定含义的内容（如～ない、できない等）。

### Deepseek阅读生成

> 日本では、コミュニケーションの際に「空気を読む」ということが非常に重視される。これは、相手が言葉にしなくても、その場の雰囲気や相手の気持ちを察して、適切に行動する能力を指す。この能力は、特にビジネスシーンや集団生活の中で不可欠なものとされている。
>
> 例えば、会議の場で自分の意見が多数派と明らかに異なる場合、あえて反対意見を主張せずに沈黙を守る人が少なくない。これは単に遠慮しているからではなく、「これ以上意見を言うと、会議の空気が悪くなる」という判断が働くからである。つまり、個人の主張よりも集団の調和を優先するという価値観が背景にある。
>
> しかし、この「空気を読む」文化は、時にストレスの原因にもなる。相手の気持ちを推し量りすぎて、自分の本当に言いたいことを言えなくなってしまうからだ。特に、若い世代の間では「本音で話せる関係」を求める声が高まっており、従来の「空気を読む」文化に違和感を覚える人も増えている。
>
> また、「空気を読めない人」、いわゆる「KY」という言葉が広く使われていることからも、この文化の浸透度がうかがえる。「KY」は明らかに否定的な意味合いを持ち、空気を読めないことは社会生活においてマイナス評価につながりかねない。
>
> 一方で、すべての場面で「空気を読む」ことが良いとは限らないという意見もある。時にはあえて空気を破ることで、新しいアイデアや改革が生まれることもあるからだ。特に、国際化が進む現代の日本では、多様な価値観を持つ人々と協力するために、「空気を読む」だけではなく、自分の意見を明確に伝える能力も同様に重要視されつつある。
>
> 結論として、「空気を読む」文化は日本のコミュニケーションの特徴の一つであり、そのメリットもデメリットも理解した上で、状況に応じて使い分けることが求められていると言えるだろう。

#### 1. ～をはじめ（として）
**要点：** 以某一代表性事物为起点，列举同类的其他事物，表「以…为首、除…之外还有」。
**接续：** 名词 + をはじめ（として）；名词 + をはじめとする + 名词
**补充：** 前项是同类事物的代表，后项必须是包含前项的复数事物，不可用于单一事物。

---

#### 2. ～からして
**要点：** 从某一个细节/例子切入，类推整体的情况，多用于评价、判断，表「从…来看就…」。
**接续：** 名词 + からして
**补充：** 后项多为负面评价，强调「仅这一点就足以说明问题」，不可接动词/形容词。

---

#### 3. ～にわたって
**要点：** 表示时间、空间、范围等跨度大，动作/状态持续覆盖整个区间，表「横跨、历时、遍及」。
**接续：** 名词 + にわたって；名词 + にわたる + 名词
**补充：** 仅用于描述大范围、长时间的客观情况，不可用于小范围、瞬时动作。

---

#### 4. ～を通じて・～を通して
**要点：** 分两种用法：A 贯穿整个期间/范围，持续同一状态；B 通过某一手段/媒介达成某事。
**接续：** 名词 + を通じて・を通して
**补充：** A用法多接「期間・年数・四季」等时间名词；B用法多接「人・組織・手段」等媒介名词，两者可互换。

---

#### 5. ～限り
**要点：** 表「尽…所能」「在…的范围内」，强调范围的全部、能力的极限。
**接续：** 名词+の / 动词辞书形/ている形 + 限り（动词多接ている形/可能动词）
**补充：** 可表「尽最大努力」，也可表「在认知/能力范围内」，后项多为意志性动作。

---

#### 6. ～だけ
**要点：** 表「到…的界限为止」「仅、只」，强调范围、程度的限定，也可表「尽可能」。
**接续：** 动词辞书形 + だけ（多接可能动词，也可接「たい/欲しい/好きな/必要な」等）
**补充：** 前后可重复同动词（如「働くだけ働く」），表「尽全力做某事」，也可表数量/程度的限定。

#### **1. ～に限り**

  * **要点：** 表示「只有这个是特别的、例外」。
  * **接续：** 名词 + に限り
  * **补充：** 这是一种硬性的书面表达（硬い言い方），常用于向公众解释规则。后项通常接「允许、适用、可以」等句子，不接否定句或表达动作意愿的句子。
      * *例：欠席理由が正当な場合に限り、出席扱いとします。*

#### **2. ～限り（は）**

  * **要点：** 只要前项的状态一直在持续，后项的状态也会一直维持（前提条件）。
  * **接续：** 普通形现在（ナ形词干+な/-である、名词+の/-である）+ 限り（は）
  * **补充：** 前后都必须是表示「状态」的词。因为它带有条件的意味，所以后半句**不能接过去式**。
      * *例：体が丈夫な限り、まだまだ山登りが楽しめるだろう。*

#### **3. ～限りでは**

  * **要点：** 将判断的依据/信息源限定在某个范围内，表「据我所及的范围来看…」。
  * **接续：** 名词+の / 动词辞书形、た形、ている形 + 限りでは
  * **补充：** 前项通常接与「获取信息」相关的动词（看、听、记得、知道、调查等）。后项是对该范围内信息的判断。
      * *例：わたしが知っている限りでは、この近所に花屋はありません。*

#### **4. ～に限って**

  * **要点：** 包含三种用法：
      * **A（例外）：** 强调「平时不这样，唯独这次例外」。
      * **B（不凑巧）：** 强调「偏偏在最不方便的时候，发生了不好的事」，表达说话人的不满。
      * **C（信任/判断）：** 因为特别相信某人或某物，所以断定其「绝对不会有坏事」。
  * **接续：** 名词 + に限って
  * **补充：** B用法常含有「运气不好」的语感；C用法后项通常接「～はずはない」「～ない」等否定形式。
      * *例A：ふだんお酒を飲まない彼が、今日に限ってかなり飲んだ。*
      * *例B：庭の手入れをしようと思っている日に限って雨が降る。*
      * *例C：うちの子に限って友達をいじめることはないと思います。*

import numpy as np    # 引入 NumPy：负责生成和计算数字
import pandas as pd   # 引入 Pandas：负责把数字做成表格（Excel）

# --- 第一步：定义“生成走路数据”的模版 ---
def generate_walking_data(duration_seconds=100):
    """
    模拟正常走路。手机在兜里会有轻微的、有节奏的晃动。
    """
    # 每一秒钟手机产生 10 个数据点（频率 10Hz）
    samples = duration_seconds * 10
    
    # 模拟 x, y, z 三个轴的震动
    # np.random.normal(中心值, 摆动幅度, 数量)
    # 这里的摆动幅度只有 0.5，说明波动很小
    x = np.random.normal(0, 0.5, samples)
    y = np.random.normal(9.8, 0.5, samples) # Y轴通常由于重力接近 9.8
    z = np.random.normal(0, 0.5, samples)
    
    # 把这些数字装进一个叫 DataFrame 的表格里
    df = pd.DataFrame({'x': x, 'y': y, 'z': z})
    
    # 贴标签：0 代表“正常”
    df['label'] = 0
    return df

# --- 第二步：定义“生成摔倒数据”的模版 ---
def generate_fall_data(num_falls=10):
    """
    模拟剧烈摔倒。加速度会瞬间产生一个巨大的“峰值”。
    """
    fall_list = []
    for _ in range(num_falls):
        # 摔倒瞬间，数值会突然从 0.5 暴增到 15 甚至更高
        impact_x = np.random.normal(0, 15, 5) # 持续 0.5 秒的冲击
        impact_y = np.random.normal(0, 15, 5)
        impact_z = np.random.normal(0, 15, 5)
        
        df = pd.DataFrame({'x': impact_x, 'y': impact_y, 'z': impact_z})
        
        # 贴标签：1 代表“摔倒”
        df['label'] = 1
        fall_list.append(df)
        
    # 把所有的摔倒片段粘在一起
    return pd.concat(fall_list)

# --- 第三步：开始正式“生产”数据 ---

# 1. 生产 1000 秒的走路数据
walking_data = generate_walking_data(1000)

# 2. 生产 50 次摔倒的数据
fall_data = generate_fall_data(50)

# 3. 把走路和摔倒的数据拼成一张超级大表
final_dataset = pd.concat([walking_data, fall_data])

# 4. 把这张大表存成电脑里的一个 CSV 文件（你可以用 Excel 打开它）
# index=False 意思是不要在左边生成多余的行号
final_dataset.to_csv('sentio_raw_data.csv', index=False)

print("Day 1 任务完成：已生成模拟数据文件 'sentio_raw_data.csv'！")

import pandas as pd
import numpy as np

# --- 1. 加载 Day 1 生产的原始账本 ---
raw_data = pd.read_csv('sentio_raw_data.csv')

# --- 2. 核心算法：算出“总力量”（合加速度） ---
# 无论手机怎么翻转，这个数值（mag）是恒定的，解决了手机朝向问题
raw_data['mag'] = np.sqrt(raw_data['x']**2 + raw_data['y']**2 + raw_data['z']**2)

# --- 3. 准备两个新筐子：X 装题目，y 装答案 ---
features = []  # 题目：这段时间的平均力量和波动程度
labels = []    # 答案：这段时间到底是 0（正常）还是 1（摔倒）

# --- 4. 滑动窗口：像放大镜一样扫描数据 ---
# window_size = 10 意味着我们每次看 10 行（大约 1 秒的数据）
window_size = 10

for i in range(0, len(raw_data) - window_size, 5): # 每次挪动 5 行，增加重叠度
    # 截取这 10 行数据
    window = raw_data.iloc[i : i + window_size]
    
    # 提取特征：把 10 个数字浓缩成 2 个核心指标
    mean_mag = window['mag'].mean()  # 这 1 秒的平均力量
    std_mag = window['mag'].std()    # 这 1 秒的波动剧烈程度（标准差）
    
    # 获取这段时间的“标准答案”
    # 如果这 10 行里有任何一行标记为 1，我们就认为这一秒发生了摔倒
    label = 1 if window['label'].any() == 1 else 0
    
    # 装进筐里
    features.append([mean_mag, std_mag])
    labels.append(label)

# --- 5. 转化为 AI 专用的格式并保存 ---
X = np.array(features)
y = np.array(labels)

# 打印一下结果，看看我们把几万行数据浓缩成了多少个“小饭团”
print(f"原始数据行数: {len(raw_data)}")
print(f"加工后的题目数量: {len(X)}")

# 保存加工后的结果，Day 3 训练 AI 就要用这两个文件了
np.save('X_features.npy', X)
np.save('y_labels.npy', y)

print("Day 2 任务完成：数据已精加工为特征矩阵！")
